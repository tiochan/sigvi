Notification filter

This form is used to define a filter for notification.
For each vulnerability properties shown below, you can indicate a value to filter. If no value is given, then there is not filter defined for this propertie.

For example, if you set "YES" at the REM field (","::", $MESSAGES["VULN_FIELD_AR_LAUNCH_REMOTELY"]) ?>), you will receive only notifications for alerts associated to vulnerabilities that can be exploided remotely.

Is important to choose the correct type of the vulnerability:

  • : if the vulnerability complains ALL the properties of the filter, then the process will continue.
  • : if the vulnerability complains ANY the properties of the filter, then the process will continue.
  • if the vulnerability complains ALL the properties of the filter, then the process will stop.
  • if the vulnerability complains ANY the properties of the filter, then the process will stop.
